# Matrix Commander — Fulfillment Pipeline Automation

## What This Is

Matrix Commander is a proof-of-concept Python tool that automates the weekly fulfillment pipeline for a food/beverage subscription company (Appy Hour). It replaces manual steps between the React allocation tool and the final RMFG production email — inventory sync, Shopify order editing (replacing Matrixify), RMFG matrix generation (replacing the portal download), gift order handling, and sheet formatting. Each proven chunk becomes a spec for the React developer to absorb into the unified tool.

## Core Value

Produce a correct, ready-to-email RMFG production sheet from Saturday morning to delivery in under 15 minutes, including gift orders — with logic clear enough for the React dev to absorb.

## Context

### Current Saturday Flow (as of April 2026)
1. Manually set inventory in Shopify (paid variant first, $0 with remainder)
2. Run React tool for **PR-CJAM only** → generates Matrixify CSV
3. Upload via Matrixify (slow) → PR-CJAM items now on orders
4. Run React tool **again** for everything else → generates second CSV
5. Upload via Matrixify again → all items now on orders
6. Download from RMFG Translator portal → manual formatting → gift merge → email

### What Matrix Commander Replaces
- Steps 1, 3, 5: `sync-shopify` (direct Shopify API, replaces slow Matrixify)
- Step 6: `generate` + `finalize` (replaces portal download + manual formatting)
- Gift handling: matrix-level child assignment (Shopify blocks edits on gift orders)

### Cut Order
- Submitted Tuesday (moved from Wednesday)
- Generated by fulfillment web app

### Key Technical Facts
- Two Shopify products per child SKU: paid variant and $0 variant
- React tool needs to see PR-CJAM already on orders before generating second pass
- Gift orders: Shopify truly blocks ALL edits, must handle at matrix/XLSX level only
- RMFG portal download format: tab "Worksheet", no ProductionDay column, needs transforms
- ~400-2500 orders per batch, ~19 unique parent SKUs, ~26 unique child SKUs

### Existing Codebase
- `matrix_commander.py` — CLI tool with all commands built but untested against live data
- `matrix_commander_web/` — Flask + pywebview web UI (port 5188)
- `appyhour/` — MCP server with Shopify/Recharge tools
- See `.planning/codebase/` for full architecture mapping

## Constraints

- **React tool is external**: Maintained by separate developer, Matrix Commander works alongside it
- **Gift orders locked in Shopify**: Cannot use order edit API, must handle at matrix level
- **Two-pass sequencing**: PR-CJAM must be uploaded before other parents (React tool dependency)
- **Live data only**: All testing must happen against real Shopify/Recharge data — no staging environment
- **pywebview + netfx**: Desktop app uses .NET Framework backend, not coreclr
- **Modular code**: Each piece must be clean enough to serve as spec for React dev
