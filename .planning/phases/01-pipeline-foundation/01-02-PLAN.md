Key implementation notes for `LeakyBucketLimiter`:
- Use `time.monotonic()` for accurate elapsed time (not `time.time()`).
- `wait(cost)` refills tokens based on elapsed time since last call: `new_tokens = elapsed * pts_per_sec`. Cap at `bucket_max`. Then if `self._tokens < cost`, sleep for `(cost - self._tokens) / pts_per_sec` seconds. Consume `cost` tokens.
- Synchronous only — no asyncio, no threading (per D-05).
- `record_response()` accepts the `extensions` dict from a Shopify GraphQL response (e.g. `resp_json.get("extensions", {}).get("cost", {})`). If `actualQueryCost` is present, deduct it from `_tokens` to keep the bucket accurate.

Key implementation notes for `shopify_retry`:
- Use `tenacity.retry` with:
  - `retry=retry_if_exception_type(RateLimitError)` — define a thin `RateLimitError(Exception)` in this module
  - `wait=wait_exponential(multiplier=1, min=2, max=60) + wait_random(0, 1)` (jitter per D-06)
  - `stop=stop_after_attempt(5)`
  - `before_sleep=before_sleep_log(logger, logging.WARNING)`
- The decorator inspects the `requests.Response` object: if `response.status_code == 429`, it extracts `Retry-After` header (if present), sleeps that duration, then raises `RateLimitError` to trigger tenacity retry.
- Add module docstring: "Shopify GraphQL rate limiter and retry policy. React equivalent: axios interceptor with token bucket + exponential backoff."
