<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI args → cmd_sync | --pass and --retry-failed come from operator; validated by argparse choices |
| web /api/sync request → app.py | pass_number from JSON body; untrusted web input |
| checkpoint file → PipelineState | JSON on disk; malformed file could cause load() to return None (safe default) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Tampering | checkpoint stage field | mitigate | PipelineState.from_dict uses PipelineStage[name] which raises KeyError on unknown values; CheckpointStore.load() catches and returns None |
| T-02-04 | Tampering | /api/sync pass_number | mitigate | Validate pass_number in [1,2] in app.py before calling cmd_sync; return 400 on invalid value |
| T-02-05 | Elevation of Privilege | DryRunGuard bypass | mitigate | DryRunGuard.assert_can_mutate() is the single enforcement point; guard instantiated in cmd_sync and passed down — cannot be bypassed by caller |
| T-02-06 | Denial of Service | sequential loop rate | accept | LeakyBucketLimiter self-calibrates; 2500 orders at 1 req/sec = ~42 min; within operator expectations for live sync |
</threat_model>
